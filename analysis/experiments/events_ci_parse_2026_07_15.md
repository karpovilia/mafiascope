## 1. Parse-rate анализ (R1-W10 / DA-9)

### 1.0. Главный факт: 98.3% «провалов парсинга» это транспортные ошибки API, а не ответы модели

Разбор `answer_raw` у всех 6 384 непарсированных проб:

| категория | n | доля провалов |
|---|---:|---:|
| `ERROR: ('Connection aborted.', ConnectionResetError(104, ...))` | 4 888 | 76.6% |
| `ERROR: HTTPSConnectionPool ... ProxyError / ConnectTimeoutError` | 1 388 | 21.7% |
| настоящий content-level отказ (модель ответила, JSON не извлечён) | 107 | 1.7% |

То есть до модели не дошли (или от неё не вернулись) 6 277 вызовов; **content-level parse rate среди доставленных ответов: 13 815 / 13 922 = 99.23%**. «Потеря 32%» это missingness инфраструктурного происхождения (сеть/прокси до api.deepseek.com в момент прогона), а не свойство поведения агентов.

### 1.3. Тесты

**Роль.** Общий parse rate: Mafia .685, Villager .680, Doctor .694. Кластерная пермутация (перестановка ролей между агентами внутри игры с сохранением набора проб каждого агента, P=4999): **p = 0.715**. Наивный chi-square (без учёта кластеризации, только как ориентир): p = 0.354. **Ролевой структуры потерь нет.**

**Раунды.** Потери концентрируются не в поздних, а в **ранних** раундах: .591 (R0) -> .986 (R3+). Per-game slope parse rate по раунду: среднее **+0.092/раунд**, отрицательный только в 9/32 игр, Wilcoxon по играм p = 1.0e-4. Транспортный error rate по раундам: 40.6% / 30.3% / 5.7% / 0.0%. Интерпретация: ошибки сети бились по конкретным прогонам, а вызовов в R0-R1 в разы больше (жив весь стол, полная батарея проб), поэтому ошибочные окна почти целиком пришлись на ранние раунды.

**EN vs RU (21/11 игр).** Pooled: EN .622, RU .782. На уровне игр (кластер = игра): EN .612 vs RU .731, Mann-Whitney **p = 0.074**, на 32 играх различие формально не значимо. Природа различия видна из межигровой дисперсии: **9 игр вообще без ошибок**, у остальных error rate 30-56% (медиана по корпусу .483, максимум .556). То есть missingness определяется сетевым состоянием конкретного прогона, а не языком как таковым.

**Content-level отказы (107 шт.).** По ролям: Mafia .9952, Villager .9904, Doctor .9927 (доля успешного парсинга среди доставленных). Chi-square p = 0.013, кластерная пермутация p = 0.028. Эффект 0.5 п.п. при 107 отказах на 13 922 ответа; направление: у мафии парсится чуть **лучше**, то есть даже этот микро-эффект не прячет мафия-сигнал. По probe_id разброс .989-.995.

### 1.5. Вывод для статьи (куда гнёт селекция)

1. Потеря 32% **не систематична по ролям** (пермутационный p = 0.715; ролевой состав parsed-выборки совпадает с полным до 0.3 п.п.) и **не связана с содержанием ответа**: 98.3% потерь возникли до получения ответа модели (сетевые ошибки), content-level parse rate 99.2%.
2. Смещение состава есть по двум осям: parsed-выборка перевзвешена к **поздним раундам** (R2+R3: 13.7% vs 9.8% в полном наборе) и к **RU-играм** (44.2% vs 38.7%), потому что ошибочные сетевые окна пришлись на ранние раунды части (преимущественно EN) прогонов.
3. Для coupling/калибровки: недопредставлены ранние, «низкоинформативные» пробы (R0 это преимущественно NO_INFO/Unknown-ответы). Возможный эффект: лёгкое завышение средней информативности/уверенности убеждений и метрик связности belief-action за счёт веса поздних раундов; ролевые сравнения (M vs V vs D) не затронуты. Потеря ведёт себя как MCAR относительно роли и содержания при фиксированном (игра, раунд), поэтому выводы Table 2 и ролевой калибровки она не подрывает; честная формулировка в статье: «probe availability is limited by transient API transport errors, orthogonal to role (permutation p = .72) and answer content (99.2% of delivered answers parse); missingness is concentrated in early rounds of a subset of runs».

## 2. CI и множественность для событийной таблицы (R1-W2, Table 2)

### 2.1. Метод

- Единица: размеченное предложение (25 392), кластер: игра (32).
- Доля роли в событии: share(e, r) = n(e, r) / n(e). Популяционные доли предложений: **Mafia .336, Villager .525, Doctor .139** (8 533 / 13 337 / 3 522).
- **CI**: cluster bootstrap, ресэмплинг целых игр с возвращением, B = 1000, seed = 0, перцентильный 95% CI.
- **p-values**: пермутация ролей внутри игры (перестановка ролевых меток между агентами игры при сохранении числа и состава предложений каждого агента), P = 4999, seed = 1; односторонняя статистика share(e, r) - pop_share(r) (популяционная доля пересчитывается в каждой пермутации).
- **Множественность**: Benjamini-Hochberg, q = 0.05, семейство из 48 сравнений (16 событий x 3 роли; OTHER вне семейства).
- **Критерий доминации ячейки**: нижняя граница 95% CI доли роли выше её популяционной доли **и** выживание BH.
